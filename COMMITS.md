# 📝 Commits by Code (CBC) Protocol

This project follows the **Commits by Code** principle. Every change is documented with intent, scope, and traceability.

## Commit Message Standard

We use **Conventional Commits** to ensure a clean, searchable history.

### Format

`<type>(<scope>): <description>`

- `chore`: Changes to the build process or auxiliary tools and libraries

## Agent Commit Guidelines

- Agents MUST use the `git-pushing` skill or equivalent smart commit script.
- Every commit should link to a `log_decision` entry if it involves an architectural shift.
- Commits should be atomic and focused on a single logical change.
