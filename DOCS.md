# 📚 Documents by Code (DBC) Protocol

This project follows the **Documents by Code** principle. Documentation is a living reflection of the system state, managed by agents.

## Chronicle Methodology

- **Architectural Logs**: Every major project decision must be recorded using the `log_decision` tool.
- **Self-Updating Briefs**: The `codeAnalysisBrief` is automatically updated on every significant PR/Commit.
- **Agent Instructions**: The `AGENT_INSTRUCTIONS.md` is the source of truth for agent behavior in this repo.

## Structure

- `/docs/chronicle/`: Decision logs and architectural evolution.
- `/docs/guides/`: User/Developer onboarding.
- `README.md`: High-level entry point (Health, Status, Tech Stack).
