# 🛡️ Security by Code (SBC) Protocol

This project follows the **Security by Code** principle. Security is not an afterthought; it is baked into the implementation.

## Core Mandates

1. **Zero-Trust Defaults**: All services, endpoints, and data access layers default to "Deny All".
2. **Schema Enforcement**: Every data structure is validated at the edge using Zod or equivalent.
3. **Secret Suppression**: No secrets in code. Use the Dashboard's BYOK (Bring Your Own Key) or Environment Variables.
4. **Audit at Rest**: All sensitive operations (Authentication, Data Mutatation) are logged to the 626Labs Audit Trail automatically.

## AI Security

- **Strict Prompting**: Agent prompts must include "Fortress Guards" to prevent prompt injection and unauthorized system access.
- **Context Pinning**: Agents are restricted to the current `projectId` context unless explicitly routed via tags.
